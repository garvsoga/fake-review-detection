Our Technology Stack

Technologies Used

Built with Python, Flask, Scikit-Learn, and modern web technologies

Python & Pandas

Core language for data preprocessing, cleaning, and ML pipeline. Pandas handles data manipulation, null removal, and text normalization.

TF-IDF Vectorization

Term Frequency–Inverse Document Frequency converts review text into numerical feature vectors that the ML model can understand and process.

Logistic Regression

A robust classification algorithm trained on 40K+ labeled reviews. Provides both binary prediction (fake/real) and probability confidence scores.

Flask REST API

Lightweight Python web framework serving the ML model via a /predict API endpoint. Handles JSON requests/responses with CORS support.

HTML / CSS / JavaScript

Modern responsive frontend with CSS animations, fetch API for backend communication, and real-time result rendering without page reloads.

Pickle Serialization

Trained model and vectorizer are saved as .pkl files using Python's pickle module, enabling instant loading without retraining.

Process

How It Works

From raw data to real-time prediction in four stages

1

Data Cleaning (clean.py)

The raw dataset (reviews.csv with 40K+ reviews) is loaded using Pandas. Null values are removed, text is lowercased, special characters are stripped, and labels are mapped from CG/OR to 1/0 (Fake/Real). Short reviews are filtered out and duplicates removed.

2

Model Training (model.py)

Cleaned data is split 80/20 for training and testing. TF-IDF Vectorizer extracts the top 5000 word features (unigrams + bigrams). A Logistic Regression classifier is trained on the TF-IDF vectors. The trained model and vectorizer are serialized as model.pkl and vectorizer.pkl.

3

Flask API (app.py)

A Flask server loads the saved model and vectorizer at startup. When a POST request hits /predict with review text, it cleans the input, transforms it to a TF-IDF vector, runs prediction, and returns a JSON response with the verdict and confidence percentage.

4

Frontend (index.html)

The user types or pastes a review in the browser. JavaScript sends the text to the Flask API using the fetch() function. The result (FAKE/REAL + confidence) is displayed instantly with visual indicators — no page reload needed.

Know Your Enemy

Types of Fake Reviews

Different fraud patterns our model is trained to detect

Paid Positive Reviews

Sellers pay individuals to write glowing 5-star reviews with generic praise and excessive superlatives.

Competitor Negative Reviews

Competitors post fake 1-star reviews to damage rival products with fabricated complaints.

AI/Computer-Generated

Machine-generated reviews created at scale. The CG (Computer Generated) label in our dataset targets these.

Incentivized Reviews

Reviews written in exchange for free products or discounts, creating biased and misleading feedback.

Review Farm Networks

Organized operations with hundreds of fake accounts systematically posting reviews across products.

Copied & Templated

Reviews copy-pasted across multiple products with only minor modifications to avoid detection.

FAQ

Frequently Asked Questions

Everything about this project

What dataset is used for training?

We use the "Fake Reviews Dataset" from Kaggle containing 40,000+ reviews labeled as CG (Computer Generated / Fake) and OR (Original / Real). The dataset covers multiple product categories including Home & Kitchen, Books, Kindle Store, and more.

What ML algorithm is used?

We use Logistic Regression from Scikit-Learn, combined with TF-IDF (Term Frequency–Inverse Document Frequency) vectorization for feature extraction. TF-IDF converts text into numerical vectors based on word importance, and Logistic Regression classifies them as fake or real.

What is TF-IDF and why is it used?

TF-IDF stands for Term Frequency–Inverse Document Frequency. It measures how important a word is in a document relative to a collection. Common words like "the" get low scores, while distinctive words get high scores. This helps the model focus on meaningful patterns that distinguish fake from real reviews.

How accurate is the model?

The Logistic Regression model achieves approximately 85-90% accuracy on the test set (20% holdout). Results vary based on preprocessing and the specific train/test split. The model also provides confidence percentages via predict_proba().

How does the frontend connect to the backend?

The frontend uses JavaScript's fetch() API to send a POST request with the review text as JSON to the Flask backend at http://127.0.0.1:5000/predict. Flask processes the text, runs it through the model, and returns a JSON response with the prediction and confidence score.

What are model.pkl and vectorizer.pkl?

These are serialized (saved) versions of the trained model and TF-IDF vectorizer using Python's pickle module. This way, we don't need to retrain the model every time the server starts — we just load the pre-trained files and make predictions instantly.
